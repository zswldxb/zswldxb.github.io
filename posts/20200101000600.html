P1006 [NOIP2008 提高组] 传纸条

题目描述

小渊和小轩是好朋友也是同班同学，他们在一起总有谈不完的话题。一次素质拓展活动中，班上同学安排做成一个 mmm 行 nnn 列的矩阵，而小渊和小轩被安排在矩阵对角线的两端，因此，他们就无法直接交谈了。幸运的是，他们可以通过传纸条来进行交流。纸条要经由许多同学传到对方手里，小渊坐在矩阵的左上角，坐标 (1,1)(1,1)(1,1)，小轩坐在矩阵的右下角，坐标 (m,n)(m,n)(m,n)。从小渊传到小轩的纸条只可以向下或者向右传递，从小轩传给小渊的纸条只可以向上或者向左传递。

在活动进行中，小渊希望给小轩传递一张纸条，同时希望小轩给他回复。班里每个同学都可以帮他们传递，但只会帮他们一次，也就是说如果此人在小渊递给小轩纸条的时候帮忙，那么在小轩递给小渊的时候就不会再帮忙。反之亦然。

还有一件事情需要注意，全班每个同学愿意帮忙的好感度有高有低（注意：小渊和小轩的好心程度没有定义，输入时用 000 表示），可以用一个 [0,100][0,100][0,100] 内的自然数来表示，数越大表示越好心。小渊和小轩希望尽可能找好心程度高的同学来帮忙传纸条，即找到来回两条传递路径，使得这两条路径上同学的好心程度之和最大。现在，请你帮助小渊和小轩找到这样的两条路径。

输入格式

第一行有两个用空格隔开的整数 mmm 和 nnn，表示班里有 mmm 行 nnn 列。

接下来的 mmm 行是一个 m×nm \times nm×n 的矩阵，矩阵中第 iii 行 jjj 列的整数表示坐在第 iii 行 jjj 列的学生的好心程度。每行的 nnn 个整数之间用空格隔开。

输出格式

输出文件共一行一个整数，表示来回两条路上参与传递纸条的学生的好心程度之和的最大值。

输入输出样例

输入 #1\#1#1

输出 #1\#1#1

说明/提示

对于 30%30\%30% 的数据，1≤m,n≤101 \leq m,n \leq 101≤m,n≤10； 对于 100%100\%100% 的数据满足：1≤m,n≤501 \le m,n \le 501≤m,n≤50。

思路

一个人如果从右下角走到左上角，等同于一个人从左上角倒着走到右下角。这样就相当于两个人同时从两人同时从 (1,1)(1,1)(1,1) 走到 (m,n)(m,n)(m,n)，我们可以想象一下，如果我们要求第 stepstepstep 步的最优结果，那么我们可以在所有的 step−1step-1step−1 步的基础上求解。当一个人走到第 stepstepstep 步时，其坐标 (i,j)(i,j)(i,j) 以满足如下条件 i+j=stepi+j=stepi+j=step。（这里为了使步数与 坐标对应，去除了第一步，可以理解为每个人的第一步为从任意一点传送到坐标 (1,1)(1,1)(1,1) 处， 2≤step≤m+n2\leq step\leq m+n2≤step≤m+n ，1≤i≤m1\leq i\leq m1≤i≤m，1≤j≤n1\leq j\leq n1≤j≤n）

那么对于每一个人来说，只用知道其行坐标 iii 以及步数 stepstepstep，就可以计算出其列坐标 jjj。

我们可以用 dpp1,p2dp_{p1,p2}dpp1,p2​ 来记录第一个人的行坐标为 p1p_{1}p1​，且第二个人的行坐标为 p2p2p2 处的最优解（请注意这里 dpdpdp 中每个元素的含义），Nx,yN_{x,y}Nx,y​ 用来表述方格图 (x,y)(x,y)(x,y) 的数。

从第 step−1step-1step−1 步到第 stepstepstep 步，只有四种，我们用 p1p1p1 和 p2p2p2 分别表示第一个人的行坐标和第二个人的行坐标：

  1. 第一个人向右走，且第二个人向右走。我们可以得出第一个人在第 step−1step-1step−1 步的列坐标为 p1p1p1，第二个人在第 step−1step-1step−1 步的列坐标为 p2p2p2。
  2. 第一个人向右走，且第二个人向下走。我们可以得出第一个人在第 step−1step-1step−1 步的列坐标为 p1p1p1，第二个人在第 step−1step-1step−1 步的列坐标为 p2−1p2-1p2−1。
  3. 第一个人向下走，且第二个人向右走。我们可以得出第一个人在第 step−1step-1step−1 步的列坐标为 p1−1p1-1p1−1，第二个人在第 step−1step-1step−1 步的列坐标为 p2p2p2。
  4. 第一个人向下走，且第二个人向下走。我们可以得出第一个人在第 step−1step-1step−1 步的列坐标为 p1−1p1-1p1−1，第二个人在第 step−1step-1step−1 步的列坐标为 p2−1p2-1p2−1。

处理过程如下：

  1. 第 stepstepstep 步，
  2. 对于第 stepstepstep 步时所有可能的 p1p1p1 取值，
  3. 对于第 stepstepstep 步时所有可能的 p2p2p2 取值，
  4. 更新 dp 数组以达到 dpp1,p2dp_{p1,p2}dpp1,p2​ 为最好的情况。

由于这里的状态转移方程会覆盖掉原来的 dpp1,p2dp_{p1,p2}dpp1,p2​ 的值。

假如紫色方框内是我们所要更新的区域 {(a,b)∣a∈p1所有的可能取值组成的集合,b∈p2所有的可能取值组成的集合}\lbrace(a,b)|a\in p1所有的可能取值组成的集合,b\in p2所有的可能取值组成的集合\rbrace{(a,b)∣a∈p1所有的可能取值组成的集合,b∈p2所有的可能取值组成的集合}。如果我们从小到大更新，则更新完蓝色块后更新红色块，这样在更新红色块的时候，用到了刚刚才更新完的蓝色块，而不是蓝色块更新前的数据，这样第 stepstepstep 步的更新使用了第 stepstepstep 步的数据，这是有问题的。解决方法是倒着更新，这样更新后的位置不会被用到，即如果我们先更新完红色块，则在第 stepstepstep 步中不会再用到红色块。

注意：我们需要考虑 p1p1p1 与 p2p2p2 取值的左端点和右端点。
