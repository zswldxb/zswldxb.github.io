P1613 跑路

题目描述

小 AAA 的工作不仅繁琐，更有苛刻的规定，要求小 AAA 每天早上在 6:006:006:00 之前到达公司，否则这个月工资清零。可是小 AAA 偏偏又有赖床的坏毛病。于是为了保住自己的工资，小 AAA 买了一个十分牛的空间跑路器，每秒钟可以跑 2k2^{k}2k 千米（kkk 是任意自然数）。当然，这个机器是用 \long\ int 存的，所以总跑路长度不能超过 max⁡ long int\max\ long\ intmax long int千米。小 AAA 的家到公司的路可以看做一个有向图，小 AAA 家为点 111，公司为点 nnn，每条边长度均为一千米。小 AAA 想每天能醒地尽量晚，所以让你帮他算算，他最少需要几秒才能到公司。数据保证 111 到 nnn 至少有一条路径。

输入格式

第一行两个整数 n,mn,mn,m，表示点的个数和边的个数。

接下来 mmm 行每行两个数字 u,vu,vu,v，表示一条 uuu 到 vvv 的边。

输出格式

一行一个数字，表示到公司的最少秒数。

输入输出样例

输入 #1\#1#1

输出 #1\#1#1

说明/提示

样例解释

1→1→2→3→41\rightarrow 1\rightarrow 2\rightarrow 3\rightarrow 41→1→2→3→4，总路径长度为 444 千米，直接使用一次跑路器即可。

数据范围

50%50\%50% 的数据满足最优解路径长度 ≤1000\leq 1000≤1000。

100%100\%100% 的数据满足 n≤50,m≤10000n\leq 50,m\leq 10000n≤50,m≤10000，最优解路径长度 ≤max⁡ long int\leq\max\ long\ int≤max long int。

思路

倍增

题目中的意思是 111 秒可以走 2kkm2^{k}km2kkm，即 4km=22km4km=2^{2}km4km=22km 只用 111 秒就可以走完，而 3km=21+20km3km=2^{1}+2^{0}km3km=21+20km 需要 222 秒走完，所以距离短花的时间并不一定少。

设 f[i][j][k]f[i][j][k]f[i][j][k] 表示从 iii 到 jjj 有一条长度为 2k2^{k}2k 的路径。如果从 iii 到 lll 有一条长度为 2k−12^{k-1}2k−1 的路径，从 lll 到 jjj 有一条长度为 2k−12^{k-1}2k−1 的路径，则我们可以得到从 iii 到 jjj 有一条长度为 2k2^{k}2k 的路径。这里我们使用 floydfloydfloyd 算法初始化所有的 111 秒能到达的路径。然后使用任意最短路算法（这里使用 floydfloydfloyd 算法）求出最短路即可。
