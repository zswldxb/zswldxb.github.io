Mask TextSpotter: An End-to-End Trainable Neural Network for Spotting Text with Arbitrary Shapes

Preface

Paper

Purpose

本文指出:

  1. 此前的大多数工作, 都是将文本检测和识别分开处理的. 然而这两个任务高度相关和互补. 一方面, 检测质量的好坏决定了识别的准确性；另一方面, 识别结果可以提供反馈, 提高检测质量.
  2. 考虑两个任务一起处理的工作, 它们不能完全以端到端的方式进行训练, 主要有两个原因:
    1. 一个是文本识别部分需要准确的位置进行训练, 而早期迭代中的位置通常不准确.
    2. 另一个是采用的 LSTM 或 CTC loss 比一般的 CNN 难优化. 而采用这些方式的文章大多是处理水平或定向文本, 还没有考虑真实场景中弯曲形状的文字.

本文提出的模型用于实现端到端(同时实现文本检测和识别)的对于任意形状的文本实例的检测与识别.

Methodology

Framework

该框架由四个部分组成: 作为主干的 FPN, 用于生成 text proposals 的 RPN, 用于边界框回归的 Fast R-CNN, 用于文本实例分割和字符分割的 mask branch. 在训练阶段, RPN 首先生成大量的text proposals, 然后将 proposal 的 RoI features 输入Fast R-CNN 和 mask branch, 生成精确的文本候选框、文本实例分割图和字符分割图.

Backbone: 基于 ResNet.

RPN: 生成大量 text proposals, 然后采用 RoI Align 提取区域特征.

Fast R-CNN: 两个任务, 一个 Box Classification 任务和一个 Box Regression 任务. 输入为 7×77\times 77×7.

Mask Branch: 两个任务, 一个 global text instance segmentation 任务和一个 character segmentation 任务. 如下图所示.

对于给定大小为 16×6416\times 6416×64 的 RoI 特征, 经过四个卷积层和一个反卷积层, 得到特征大小为 32×128×25632\times 128\times 25632×128×256 的输入, 其中 256256256 为通道数, 32×12832\times 12832×128 为长宽, 将这个特征用于最后的卷积层, 用于预测任务, 维度为 32×128×3832\times128\times3832×128×38, 一个任务为文本实例分割, 另一个任务为字符分割, 其中维度为 37=26(字母)+10(数字)+1(背景)37=26(字母)+10(数字)+1(背景)37=26(字母)+10(数字)+1(背景).

Inference

与在训练阶段使用来自 RPN 的输出 RoIs 作为 mask branch 的输入不同, 在推理阶段, 我们使用 Fast R-CNN 的输出作为 proposal 以产生预测的 global maps 和 character maps, 因为 Fast R-CNN 的输出更准确.

对于测试图片 III, 经过 Fast R-CNN 并通过 NMS 过滤掉冗余的候选框后, 将剩余的 proposal 输入branch 以生成 global maps 和 character maps, 并在 global map 上直接计算文字区域的轮廓的得到预测 polygons, 对 character maps 采用 Pixel Voting 算法得到字符序列.

Pixel Voting

首先对 background map 二值化, 对于取值为 0−2550-2550−255 background map 设置阈值为 192192192, 大于阈值为 111, 小于为 000. 然后计算在每一个位置上的所有 character map 的平均值, 这个值被设为每个位置上的每一字符类的概率. 将最大平均值的字符类指定给区域, 然后从左向右对所有字符分组.
