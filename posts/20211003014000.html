Pytorch.torch.nn.MultiheadAttention

网络层 MultiheadAttention 为多头注意力机制. 其过程如下:


图 1

Pytorch 的官方实现对多种情况都进行了处理, 这里首先介绍 _in_projection(), _in_projection_packed() 函数, 这两个函数实现对 Q,K,VQ,K,VQ,K,V 的仿射变换, 然后介绍 _scaled_dot_product_attention() 函数, 该函数实现注意力机制, 最后介绍 multi_head_attention_forward() 函数, 该函数实现了多头注意力机制.

_in_projection() 函数

  • q: query 张量.
  • k: key 张量.
  • v: value 张量.
  • w_q: 对 q 的仿射变换权重张量.
  • w_k: 对 k 的仿射变换权重张量.
  • w_v: 对 v 的仿射变换权重张量.
  • b_q: 对 q 仿射变换偏置张量.
  • b_k: 对 k 仿射变换偏置张量.
  • b_v: 对 v 仿射变换偏置张量.

_in_projection() 函数实现 q, k, v 的仿射变换(注意在 multi_head_attention_forward 中进行了维度变换, 这一细节在 multi_head_attention_forward 部分介绍, 本小节中使用的维度数仅在本小节有效, 与其他小节的维度可能名字相同, 但数值可能不同).

设 Q∈RSQ×NQ×EQQ\in\R^{S_{Q}\times N_{Q}\times E_{Q}}Q∈RSQ​×NQ​×EQ​, K∈RSK×NK×EKK\in\R^{S_{K}\times N_{K}\times E_{K}}K∈RSK​×NK​×EK​, V∈RSV×NV×EVV\in\R^{S_{V}\times N_{V}\times E_{V}}V∈RSV​×NV​×EV​, WQ∈REQ×EQW^{Q}\in\R^{E_{Q}\times E_{Q}}WQ∈REQ​×EQ​, BQ∈REQB^{Q}\in\R^{E_{Q}}BQ∈REQ​, WK∈REQ×EKW^{K}\in\R^{E_{Q}\times E_{K}}WK∈REQ​×EK​, BK∈REQB^{K}\in\R^{E_{Q}}BK∈REQ​, WV∈REQ×EVW^{V}\in\R^{E_{Q}\times E_{V}}WV∈REQ​×EV​, BV∈REQB^{V}\in\R^{E_{Q}}BV∈REQ​, 其中 Q,K,VQ,K,VQ,K,V 的维度含义为序列维度, 批量维度和特征维度. 而该函数将 Q,K,VQ,K,VQ,K,V 的特征维度数均变为 EQE_{Q}EQ​.

Q′=QWQT+BQK′=KWKT+BKV′=VWVT+BV\begin{aligned} Q^{'}=&Q{W^{Q}}^{T}+B^{Q}\\ K^{'}=&K{W^{K}}^{T}+B^{K}\\ V^{'}=&V{W^{V}}^{T}+B^{V}\\ \end{aligned}Q′=K′=V′=​QWQT+BQKWKT+BKVWVT+BV​

_in_projection_packed() 函数

  • q: query 张量.
  • k: key 张量.
  • v: value 张量.
  • w: 仿射变换权重张量.
  • b: 仿射变换偏置张量.

_in_projection_packed() 函数实现 q, k, v 的仿射变换, 其中权重张量和偏置张量为被打包的张量. 其实现的功能与 _in_projection() 相同.

_scaled_dot_product_attention() 函数

  • q: query 张量.
  • k: key 张量.
  • v: value 张量.
  • attn_mask: attention_mask 张量. 需要被忽略的位置上的数值为 -inf.
  • dropout_p: 若 0<dropout_p≤10<\text{dropout\_p}\leq 10<dropout_p≤1, 则在计算过程中使用 Dropout\text{Dropout}Dropout 网络层.

_scaled_dot_product_attention() 函数实现 Scaled Dot-Product Attention(注意在 multi_head_attention_forward 中进行了维度变换, 这一细节在 multi_head_attention_forward 部分介绍, 本小节中使用的维度数仅在本小节有效, 与其他小节的维度可能名字相同, 但数值可能不同).

multi_head_attention_forward() 函数

  • query: query 张量.
  • key: key 张量.
  • value: value 张量
  • embed_dim_to_check: 嵌入特征空间维度数, 该参数用于检测 query.shape[-1] 是否等于 embed_dim_to_check, 不等于时报错.
  • num_heads: 多头注意力中头的个数.
  • in_proj_weight: 输入特征仿射变换权重张量, 若不为 None, 则为 WI∈R(3⋅E)×EW^{I}\in\R^{(3\cdot E)\times E}WI∈R(3⋅E)×E.
  • in_proj_bias: 输入特征仿射变换偏置张量, 若不为 None, 则为 BI∈R3⋅EB^{I}\in\R^{3\cdot E}BI∈R3⋅E.
  • bias_k: 对 key 的序列维度增加偏置, 若不为 None, 则为 BK∈R1×1×EB^{K}\in\R^{1\times1\times E}BK∈R1×1×E.
  • bias_v: 对 value 的序列维度增加偏置, 若不为 None, 则为 BV∈R1×1×EB^{V}\in\R^{1\times1\times E}BV∈R1×1×E.
  • add_zero_attn: 是否沿序列维度增加额外的零注意力.
  • dropout_p: 若 0<dropout_p≤10<\text{dropout\_p}\leq 10<dropout_p≤1, 则在计算过程中使用 Dropout\text{Dropout}Dropout 网络层.
  • out_proj_weight: 输出特征仿射变换权重张量, 若不为 None, 则为 WO∈RE×EW^{O}\in\R^{E\times E}WO∈RE×E.
  • out_proj_bias: 输出特征仿射变换偏置张量, 若不为 None, 则为 BO∈REB^{O}\in\R^{E}BO∈RE.
  • training: training=True, 则为训练阶段, training=False, 则为测试阶段, 此时将 dropout_p 置为 000.
  • key_padding_mask: 若不为 None, 则为 key_padding_mask∈RN×S\text{key\_padding\_mask}\in\R^{N\times S}key_padding_mask∈RN×S, key_padding_maski,j=True\text{key\_padding\_mask}_{i,j}=\text{True}key_padding_maski,j​=True, 则忽略 key\text{key}key 的第 iii 个序列的第 jjj 个位置, key_padding_maski,j=False\text{key\_padding\_mask}_{i,j}=\text{False}key_padding_maski,j​=False, 则不做任何改变.
  • need_weights: 输出时是否需要输出注意力(Attn\text{Attn}Attn)张量.
  • attn_mask: 掩码张量, 2D 张量 RL×S\R^{L\times S}RL×S, 或 3D 张量 R(N⋅num_heads)×L×S\R^{(N\cdot\text{num\_heads})\times L\times S}R(N⋅num_heads)×L×S.
  • use_separate_proj_weight: use_separate_proj_weight=False 时, in_proj_weight 和 in_proj_bias 有效, q_proj_weight, k_proj_weight, v_proj_weight 无效, use_separate_proj_weight=True 时, q_proj_weight, k_proj_weight, v_proj_weight 有效, in_proj_weight 和 in_proj_bias 无效.
  • q_proj_weight: query 仿射变换权重张量, 若不为 None, 则为 WQ∈RE×EW^{Q}\in\R^{E\times E}WQ∈RE×E.
  • k_proj_weight: key 仿射变换权重张量, 若不为 None, 则为 WK∈RE×EKW^{K}\in\R^{E\times E_{K}}WK∈RE×EK​.
  • v_proj_weight: value 仿射变换权重张量, 若不为 None, 则为 WV∈RE×EVW^{V}\in\R^{E\times E_{V}}WV∈RE×EV​.
  • static_k: static_k 不为 None 时使用固定的 key∈R(N⋅num_heads)×S×Enum_heads\text{key}\in\R^{(N\cdot\text{num\_heads})\times S\times \frac{E}{\text{num\_heads}}}key∈R(N⋅num_heads)×S×num_headsE​.
  • static_v: static_v 不为 None 时使用固定的 value∈R(N⋅num_heads)×S×Enum_heads\text{value}\in\R^{(N\cdot\text{num\_heads})\times S\times \frac{E}{\text{num\_heads}}}value∈R(N⋅num_heads)×S×num_headsE​.

其中 NNN 为小批量样本数, SSS 为源序列长度, LLL 为目标序列长度, EEE 表示嵌入特征维度数.

multi_head_attention_forward() 函数实现多头注意力机制.

第 111 步: 检验特征维度数 EEE, 检验特征维度数 EEE 可以被头数 num_heads\text{num\_heads}num_heads 整除为 head_dim\text{head\_dim}head_dim, 根据 use_separate_proj_weight 检验 key 和 value 的维度数是否对应. 结束后 Q∈RL×N×EQ\in\R^{L\times N\times E}Q∈RL×N×E, K∈RS×N×EKK\in\R^{S\times N\times E_{K}}K∈RS×N×EK​, V∈RS×N×EVV\in\R^{S\times N\times E_{V}}V∈RS×N×EV​, 头数为 num_heads\text{num\_heads}num_heads, 每个头的特征数为 head_dim\text{head\_dim}head_dim, 总的特征数为 EEE.

第 222 步: 根据 use_separate_proj_weight 和 in_proj_size, 对 query, key, value 进行仿射变换得到 Q∈RL×N×EQ\in\R^{L\times N\times E}Q∈RL×N×E, K,V∈RS×N×EK,V\in\R^{S\times N\times E}K,V∈RS×N×E.

第 333 步: 若 attn_mask 不为 None, 检验 attn_mask 的维度和数据类型, 若其为 2D2D2D 张量, 则增加第 000 维为样本维度. 结束后 attn_mask∈R1×L×S\text{attn\_mask}\in\R^{1\times L\times S}attn_mask∈R1×L×S 或 attn_mask∈R(N⋅num_heads)×L×S\text{attn\_mask}\in\R^{(N\cdot \text{num\_heads})\times L\times S}attn_mask∈R(N⋅num_heads)×L×S(为了描述方便, 后续仅考虑 attn_mask∈R(N⋅num_heads)×L×S\text{attn\_mask}\in\R^{(N\cdot \text{num\_heads})\times L\times S}attn_mask∈R(N⋅num_heads)×L×S 的情况).

第 444 步: 若 key_padding_mask 不为 None, 检验 key_padding_mask 的数据类型.

第 555 步: 若 bias_k 和 bias_v 不为 None, 且 static_k 和 static_v 均为 None, 则对 KKK, VVV 沿序列维度(此时为第 000 维)填充偏置, 并对 attn_mask 和 key_padding_mask 的源序列维度(最后一个维度)的最后填充 000. 结束后 K,V∈RS′×N×EK,V\in\R^{S^{'}\times N\times E}K,V∈RS′×N×E, attn_mask∈R(N⋅num_heads)×L×S′\text{attn\_mask}\in\R^{(N\cdot \text{num\_heads})\times L\times S^{'}}attn_mask∈R(N⋅num_heads)×L×S′, key_padding_mask∈RN×S′\text{key\_padding\_mask}\in\R^{N\times S^{'}}key_padding_mask∈RN×S′, S′=S+1S^{'}=S+1S′=S+1. 对于其他情况, S′=SS^{'}=SS′=S.

第 666 步: 对 Q,K,VQ,K,VQ,K,V 的维度进行变换(具体变换过程参考源码), 变换为 Q∈R(N⋅num_heads)×L×head_dimQ\in\R^{(N\cdot\text{num\_heads})\times L\times\text{head\_dim}}Q∈R(N⋅num_heads)×L×head_dim, K,V∈R(N⋅num_heads)×S′×head_dimK,V\in\R^{(N\cdot\text{num\_heads})\times S^{'}\times\text{head\_dim}}K,V∈R(N⋅num_heads)×S′×head_dim.

第 777 步: 如 add_zero_attn=True, 则对 KKK, VVV 沿序列维度(此时为第 111 维)填充零张量, 并对 attn_mask 和 key_padding_mask 的源序列维度(最后一个维度)的最后填充 000. 结束后 K,V∈R(N⋅num_heads)×S′′×head_dimK,V\in\R^{(N\cdot\text{num\_heads})\times S^{''}\times\text{head\_dim}}K,V∈R(N⋅num_heads)×S′′×head_dim, attn_mask∈R(N⋅num_heads)×L×S′′\text{attn\_mask}\in\R^{(N\cdot \text{num\_heads})\times L\times S^{''}}attn_mask∈R(N⋅num_heads)×L×S′′, key_padding_mask∈RN×S′′\text{key\_padding\_mask}\in\R^{N\times S^{''}}key_padding_mask∈RN×S′′, S′′=S′+1S^{''}=S^{'}+1S′′=S′+1. 对于其他情况, S′′=S′S^{''}=S^{'}S′′=S′.
