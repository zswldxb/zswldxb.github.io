Faster R-CNN: Towards Real-Time Object Detection with Region Proposal Networks

Preface

Paper

Purpose

本文指出: 目前的目标检测网络主要的瓶颈在于 region proposal algorithm. 在R-CNN中, 采用 Selective Search(SS)的 region proposal algorithm, 其算法速度较慢, 且 SS 是在 CPU 上实现的, 不能充分利用 GPU 加速.

本文提出的模型采用了最先进的 Region Proposal Networks(RPNs), 将 RPN 与 Fast-RCNN 相结合, 实现端到端的目标检测.

(a)首先构建 Pyramids of images 和 feature map, 再对不同尺度下的特征进行分类.

(b)在 feature map 上运行不同大小的 filter.

Methodology

Framework

本文的目标检测框架称为 Faster R-CNN, 由两个模块组成.

  1. 生成 proposal 的深度全卷积网络.
  2. 使用 proposal 的 Fast R-CNN 检测器.

Region Proposal Networks

Backbone: Zeiler and Fergus model 或 Simonyan and Zisserman model(VGG-16), 使用这些网络进行特征提取, 输出 feature map.

对于使用 Backbone 提取的 feature map, 将其经过 n×nn\times nn×n(本文采用 n=3n=3n=3)的卷积核(with ReLU following)进行一次特征提取, 输出维度为 256256256-d(ZF)和 512512512-d(VGG), 将提取后的特征输入两个 fully-connected layers: 一个 box-regression layer(reg)和一个 box-classification layer(cls). 由于共享feature map, 因此, reg 和 cls 前使用 1×11\times 11×1 再次分别提取特征.

Anchors

对于经过 Backbone 提取后的特征 C×H×WC\times H\times WC×H×W, 假设我们采用的 anchor 生成方式的最大数目为 kkk(如采用 333 种不同大小, 333 种不同的横纵比, 则 anchor 生成方式的最大数目为 3×33\times 33×3), 则对于每个位置 (i,j)(i,j)(i,j)(0≤i<H,0≤j<W0\leq i<H,0\leq j<W0≤i<H,0≤j<W), 生成 anchor 的最大数目为 kkk, 而对于每一个 anchor, 可以用 (x,y,w,h)(x,y,w,h)(x,y,w,h) 的四元组表示, 其中 (x,y)(x,y)(x,y) 表示 anchor 的中心坐标, www 表示 anchor 的宽度, hhh 表示 anchor 的高度. 而每一个 anchor, 使用两个数表示其 object or not
object 的概率, 因此对于每一个位置 (i,j)(i,j)(i,j), reg layer 需要 4k4k4k 个输出, cls layer 需要 2k2k2k 个输出. 因此理论上对于一个 feature map, 其 anchor 的总数为 kHWkHWkHW. 此时对于 VGG, 其 RPN 的参数数量为 3×3×512×512+512×(4+2)×9=2.4×1063\times 3\times 512\times 512 +512\times(4+2)\times 9=2.4\times 10^{6}3×3×512×512+512×(4+2)×9=2.4×106, 少于 MultiBox proposal layer.

Translation-Invariant Anchors

本文中一个重要特性是它是平移不变的, 无论是在锚点还是计算相对于锚点的建议函数.

Multi-Scale Anchors as Regression References

本文在同一个位置生成多个 anchor.

Loss Function

For training RPNs, we assign a binary class label (of being an object or not) to each anchor.

本文采用两种启发式方法生成 positive label:

  1. the anchor/anchors with the highest Intersection-over-Union (IoU) overlap with a ground-truth box.
  2. an anchor that has an IoU overlap higher than 0.7 with any ground-truth box.

因此 RPN 的损失函数为:

对 anchor 位置和大小微调

对于某 mini-batch 中第 iii 个anchor, 其位置可以表示为 (xa,ya,wa,ha)(x_{a},y_{a},w_{a},h_{a})(xa​,ya​,wa​,ha​), 而与其对应的 ground-true 的位置为 (x∗,y∗,w∗,h∗)(x^{*},y^{*},w^{*},h^{*})(x∗,y∗,w∗,h∗), 假设其经过 reg layer 预测的 box 为 (x,y,w,h)(x,y,w,h)(x,y,w,h), 则有真实的变化量为 t∗t^{*}t∗, 模型微调后的 box 的变化量为 ttt, 可以得到如下关系:

This can be thought of as bounding-box regression from an anchor box to a nearby ground-truth box.

Training RPNs

由于负类较多, 本文采用对同一个图像, 随机采样 256256256 个 anchor, 尽量使得正类与负类比例为 1:11:11:1.

Sharing Features for RPN and Fast R-CNN

本文讨论了三种训练整个模型的方式:

  1. Alternating training, 首先训练 RPN, 在使用生成的 proposal 训练 Faster R-CNN, 在使用 Faster R-CNN 微调 RPN, 并重复以上过程直到收敛(固定一个训练另一个).
  2. Approximate joint training, 将 RPN 和 Faster R-CNN 合并为一个网络, 联合训练.
  3. Non-approximate joint training. 不讨论.

Implementation Details

对于边界的 anchor 需要小心处理.

由于 RPN 的 proposal 高度重叠, 所以对 cls 采用 NMS(非极大值抑制).
