配置npm在安装全局模块时的路径和缓存cache的路径

因为在执行例如 npm install webpack -g 等命令全局安装的时候，默认会将模块安装在 C:\Users\用户名\AppData\Roaming 路径下的 npm 和 npm_cache 中，不方便管理且占用 C 盘空间，所以这里配置自定义的全局模块安装目录，在 node.js 安装目录下新建两个文件夹 node_global 和 node_cache，如图所示：

执行完后，配置环境变量，如下：

  • “环境变量” -> “系统变量”：新建一个变量名为 NODE_PATH， 值为 D:\tools\nodejs\node_global\node_modules。

  • “环境变量” -> “用户变量”：编辑用户变量里的 Path，将相应 npm 的路径 C:\Users\用户名\AppData\Roaming\npm 改为 D:\tools\nodejs\node_global。

配置完成。
